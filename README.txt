# Exotic Flora — интернет-магазин экзотических растений

Exotic Flora — дипломный веб-проект: многостраничный интернет-магазин с PHP/MySQL backend, пользовательскими аккаунтами, корзиной в базе данных, оформлением заказов и админ-панелью для управления товарами.

## Структура проекта

```
/frontend
  index.html          Главная страница
  catalog.html        Каталог товаров с фильтрами, поиском и AJAX-корзиной
  checkout.html       Оформление заказа
  my_orders.php       Личный кабинет пользователя с текущими и завершёнными заказами
  login.html          Авторизация
  register.html       Регистрация
  blog.html           Страница советов
  contact.html        Контакты
  style.css           Общие стили
  script.js           Клиентская логика и AJAX-запросы
  images/             Изображения сайта

/backend
  helpers.php         Сессии, валидация, JSON-ответы, защита страниц
  /admin
    index.php         Админ-панель CRUD товаров
    orders.php        Админ-страница управления заказами
    admin.js          AJAX-логика админ-панели

/config
  database.php        PDO-подключение к MySQL

/api
  auth.php            Регистрация, вход, выход, текущий пользователь
  products.php        Получение каталога
  cart.php            Добавление, изменение количества и удаление из корзины
  order.php           Оформление заказа
  my_orders.php       API личного кабинета пользователя
  admin_products.php  API админ-панели для CRUD товаров
  admin_orders.php    API админ-панели для смены статусов заказов

/database
  schema.sql          Создание базы, таблиц, связей, стартовых товаров и администратора
```

## Запуск локально через OpenServer/XAMPP

1. Скопируйте проект в папку веб-сервера:
   - OpenServer: `domains/exotic-flora`
   - XAMPP: `htdocs/exotic-flora`
2. Запустите Apache и MySQL.
3. Откройте phpMyAdmin.
4. Импортируйте файл `database/schema.sql`. Он создаст базу `exotic_flora`, таблицы и стартовые товары.
5. Проверьте настройки подключения в `config/database.php`:
   - `DB_HOST = localhost`
   - `DB_NAME = exotic_flora`
   - `DB_USER = root`
   - `DB_PASSWORD = ''` для стандартного XAMPP/OpenServer без пароля
6. Откройте сайт:
   - `http://localhost/exotic-flora/frontend/index.html`
   - каталог: `http://localhost/exotic-flora/frontend/catalog.html`
   - админ-панель товаров: `http://localhost/exotic-flora/backend/admin/index.php`
   - админ-панель заказов: `http://localhost/exotic-flora/backend/admin/orders.php`

## Тестовые данные

Администратор создаётся SQL-файлом:

- email: `admin@exoticflora.ru`
- пароль: `password`

Обычных пользователей можно создавать на странице `frontend/register.html`.

## Основные функции

### Регистрация и авторизация

`api/auth.php` принимает AJAX-запросы для регистрации, входа и выхода. Пароль сохраняется только в виде хеша через `password_hash()`, проверяется через `password_verify()`. После успешного входа создаётся PHP-сессия, а интерфейс показывает email пользователя и кнопку выхода.

### Каталог товаров

`api/products.php` отдаёт товары из таблицы `products`. На странице `frontend/catalog.html` товары загружаются через `fetch`, после чего работают фильтры по категориям, поиск по названию и описанию, сортировка по цене/названию и пагинация.

### Корзина

Корзина больше не хранится в `localStorage`. Все операции идут через `api/cart.php` и таблицу `cart_items`. Каждая позиция привязана к `user_id`, поэтому у каждого пользователя своя корзина. Поддерживаются добавление товара, изменение количества и удаление. Изменение количества работает без перезагрузки страницы.

### Оформление заказа

`frontend/checkout.html` показывает содержимое корзины из базы данных. При подтверждении формы `api/order.php` создаёт запись в `orders` со статусом `Новый`, переносит товары в `order_items`, считает итоговую сумму и очищает корзину. Операция выполняется в транзакции, чтобы заказ и очистка корзины были согласованы.

### Админ-панель

`backend/admin/index.php` доступна только пользователям с ролью `admin`. В панели можно добавлять, редактировать и удалять товары. Запросы отправляются без перезагрузки через `api/admin_products.php`, затем список обновляется. Страница `backend/admin/orders.php` показывает все заказы и позволяет мгновенно менять статус через `api/admin_orders.php`.

### Личный кабинет

`frontend/my_orders.php` показывает заказы текущего пользователя из `api/my_orders.php`. Заказы разделены на вкладки: текущие (`Новый`, `В обработке`, `Отправлен`) и история (`Доставлен`, `Отменён`), отсортированы от новых к старым.

## Безопасность

- SQL-инъекции предотвращаются через PDO prepared statements во всех запросах с пользовательскими данными.
- Пароли не хранятся открытым текстом: используется `password_hash()` и `password_verify()`.
- Доступ к корзине и заказам требует активной сессии пользователя.
- Доступ к админке проверяется по роли `admin`.
- Вывод пользовательских и товарных данных на PHP-страницах экранируется функцией `e()` на базе `htmlspecialchars()`.
- Данные, вставляемые JavaScript в DOM, проходят через `escapeHTML()`.
- Формы валидируются на клиенте и повторно на сервере.
- Ошибки API возвращаются в едином JSON-формате без раскрытия внутренних деталей сервера.

## База данных

Основные таблицы:

- `users` — пользователи, email, хеш пароля, роль.
- `products` — товары каталога.
- `cart_items` — позиции корзины, связанные с пользователем и товаром.
- `orders` — заказы пользователя с контактными данными, доставкой, итоговой суммой и статусом.
- `order_items` — состав заказа с количеством и ценой товара на момент покупки.

Связи описаны foreign keys в `database/schema.sql`.
